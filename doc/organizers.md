# Documentation for organizers of a Teddy Bear Hospital

If you are an organizer of an instance of the Teddy Bear Hospital and you also want to have an "X-Ray machine" with our software, you need to ensure the following:

* Find a technical person that understands the [Documentation for system administators](sysadmins.md) and can help you with that. This might be you, but it might also require more technical skills.
* Clarify with that person whether you have access to the required server hardware.
* In preparation you should have access to the following additional hardware:
	* A printer
	* Sticky label paper to feed into a normal printer
* Organize hardware for the on-sight X-Ray machine. The minimum configuration is:
	* One consumer PC or Laptop (no specific requirements)
	* A webcam (external)
	* Constant Internet access on-sight that allows you to communicate with the server backend
* A second webcam is recommended to separate the QR Code scanner from the actual X-Ray
* Additional monitors might be helpful for showing results to the patients, as well as showing a slideshow of recently generated images to bypassers.
* Build a proper "X-Ray" machine (e.g. from a cardboard box). While not technically required, this
	* adds to the illusion
	* enables robust lighting conditions
	* You can have multiple cushions inside that help you place animals properly
* Print QR Codes by switching to "QR-Code generation" in the application footer
* Customize the files in [`assets/parents`](https://github.com/ssciwr/tbk-app/tree/main/assets/parents) that will be stored for all parents in the cloud storage
* Make sure that all your "doctors" are familiar with the UI e.g. by reading the [documentation for doctors](doctors.md)
